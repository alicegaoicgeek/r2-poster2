# 有了 50 次预算，怎么花最值？

R2-Cov · 回归筛选 · 03 / 06

Poster 02 回答了「为什么只跑 50 个」。这一张回答：效率不只一条路。运行效率 × 搭建效率。

## 两条效率路径

### 路径一 · 运行效率

Agent 动态筛选，每次 CI 只跑 Top-N 个最相关测试。

独立验证：速度最高 **6.6x**，捕获全部失败。

### 路径二 · 搭建效率

标准接口复用 env，新模块接入：两周 → 半天。

env 骨架不变，只改 DUT 连接和 C 模型调用。

同构测试 → 可比性更强 → Agent 筛选冗余更容易。

---

某算法模块，RTL 设计两周 → 验证半天搭好 env，回归 4h → 20min。总周期从 16 天 → 3 天。不是某一项快了，是两项叠加相乘。

---

**筛选解决运行效率，标准接口解决搭建效率。双轮驱动，效果相乘。**
