# R2-四件套：Skill会跑，产线不会跑

你的 AI 能跑通 RTL 模块。但你的产线跑不通。  
90% 断裂不在单点能力，在点与点之间的衔接。  
根因不在模型强弱，在你缺了操作系统层。

**R2-AIOS = Harness × Oracle × Memory × Loop**  
四件套之间是 ×，不是 +。任一项为零，整体为零。  
产线上限由最弱那一项决定。

## 四件套

### Harness 编排引擎
确定性编排引擎  
agent / parallel / pipeline / while  
编排不靠对话逐轮调度，靠代码

### Oracle 机械裁判
机械裁判 — 不是 LLM  
用模型裁模型 = 幻觉加速器  
写 RTL / 判错 / 修复 三者分离  
判据：同一件事问智能体两次，就算失败

### Memory 持久记忆
跨会话持久化  
代码是唯一文档  
记忆不让从零开始

### Loop 自动收敛
产线跑到机械条件满足才停  
不靠人感觉判断

## 产线总线（系统底座）
调度 | 隔离 | 持久化 | 收敛 | 审计  
操作系统层决定产线能否稳定、可控、可交付

## 停止条件 · 四条硬线
1. 连续 3 轮零发散
2. 全绿签名一致
3. DRC clean + WNS ≥ 0
4. 人签核落盘

→ **验收闸门**：四条全部通过，产线才算完成

## 缺一项会怎样
- 有 Harness 没 Oracle → 高速幻觉加速器
- 有 Oracle 没 Memory → 每天雇新实习生从零开始
- 有 Memory 没 Loop → 知识在涨，但不知何时交付

Claude Code 是应用程序，跑在裸机上。  
没有调度、没有保护、没有持久化、没有收敛。  
直接接进芯片产线 = 概率性进程直接写裸设备。

「我让 AI 碰我的 RTL，它会不会悄悄害死我的流片？」  
回答不在模型能力上，在操作系统层上。
