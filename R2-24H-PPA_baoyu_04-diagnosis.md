# R2-诊断与三三制：有信息量的候选从哪里来

04/09

## 诊断流程

1. **取数** — 从 timing 报告提取 10~25 条关键路径（.rpt）
2. **诊断** — 逐条分析路径，写清根因（六类之一）
3. **分工** — 按根因映射三路策略，进入三三制并行优化

- 根因写清三路才有分工；只写“不够快”则三路同构
- 连续两圈根因不变且 score 无提升，先复审诊断

**关键路径（每轮）**：下限 10 | 上限 25

## 六种根因（写清根因，才能有效分工）

| # | 根因 |
|---|------|
| 01 | 组合逻辑深度过大 |
| 02 | 控制信号高扇出 |
| 03 | 标志更新锥过深 |
| 04 | 选择信号过晚参与运算（sel） |
| 05 | 长表达式可拆分（A+B+C+D+E+… → 分组求和） |
| 06 | FSM 输出拖长下游（组合逻辑很长） |

## 反模式

- 根因只写“不够快” → 三路变成同一改法三种参数
- 诊断与根因—策略对应写入 INDEX → 形成可复盘、可复用的知识资产
