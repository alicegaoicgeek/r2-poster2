# R2-Loop Engineering = 敏捷操作系统

**扩展同构：** Loop 之于 AI Agent，等于 Scrum 之于人类开发团队。  
**两者都是整合框架，不是单项技术。** 两者都不是发明新能力，而是把已有实践拼成节奏。  
**共同质量上限：** 不取决于机制本身，取决于完成标准定义得多硬。

## 对照表

| 人类敏捷世界 | 内核 / 完成标准 | Agent 敏捷世界 |
|-------------|----------------|---------------|
| Sprint 节奏驱动，或目标驱动 | 内核 / 完成标准 (Completion Standard) | /loop 或 /goal 节奏驱动，或目标驱动 |
| Scrum-of-Scrums 多团队并行 | | Dynamic Workflow 多团队并行，变成 Agent 扇出 + 扇入 |
| CI/CD Pipeline 自动检查，自动触发，自动拦截 | | Automations + Hooks 自动检查，自动触发，自动拦截 |
| Wiki / Confluence / ADR 共享上下文，沉淀到可调用资产 | | Skills / CLAUDE.md / Memory 共享上下文，沉淀到可调用资产 |
| 分支策略 / 环境隔离 并行试错，不污染主线 | | Git worktree 并行试错，不污染主线 |

## 一句话
**Scrum 管人类协作的节奏。  
Loop Engineering 管 Agent 协作的节奏。**
