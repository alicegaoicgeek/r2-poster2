过程层：合同式 prompt 怎么管 64 agent

HOW · PROCESS
  • 钉死完成：定义图/桥/圈/CDC；禁止立方、平面等额外假设
  • 黑名单：特殊图类、非恰好 2、归约未证、固定规模、假反例 → 不算
  • 多样 portfolio：代数 / 结构 / 流 / 嵌入 / 极值 / 计算自检
  • 早期独立：不把「当前最看好」告知多数 agent
  • Registry：按数学思想分组；同族挤堆则分流
  • Blocked：卡在与原题同强引理 → 停烧，除非新机制
  • Adversarial：恰好两次、假圈、割点、循环用 CDC 本身
  • 回传：只收引理/构造/方程/反例；拒收「有进展」
