# Fable 5 安全路由架构：性能与安全的动态平衡之道

通过三级安全分类器在性能效率和安全防护间动态风险层间，并基于风险等级动态路由到 Fable 5 或安全性更高的 Claude Opus 4.8，实现 Mythos 赋能在大众市场的安全释放。

## 整体架构概览

**用户请求**（文本/图像/代码等）
→ **三级安全分类器路由层**（独立运行）
1. 网络安全分类器
2. 生物/化学安全分类器
3. 模型滥用防护分类器

- 任一分类器触发风险阈值 → 触发降级机制
- **低风险**（+ 直通）→ Claude Fable 5（Mythos 赋能）主流程处理 → 返回结果
  - 快速无中断
  - 无状态变更提示
  - 高吞吐与安全保障
- **高风险**（+ 降级）→ Claude Opus 4.8（安全稳定型）降级处理

## 1. 三级安全分类器的工作机制

### ① 网络安全分类器（第一道屏障）
检测涉及网络攻击或信息窃取技术的直接利用或开发意图
- 恶意软件、漏洞利用工具、攻击脚本编写或优化
- 入侵思路/漏洞扫描/钓鱼方案
- 渗透测试方案
- 其他恶意渗透、破坏或未授权访问的请求

### ② 生物/化学安全分类器（第二道屏障）
检测涉及生物或化学制品高风险技术的合成、改进、使用
- 致命性生物物质、化学毒素合成技术方案
- 大规模杀伤性制剂制备流程
- 双重用途研究类请求

### ③ 模型滥用防护分类器（第三道屏障）
防御针对模型本身的滥用或盗取攻击
- 系统化大量调用尝试提取模型权重、架构细节
- 训练数据样本盗取
- 核心能力逆向及反向推断

**语义 proximity 风险分级机制**：基于先验知识理解，判断请求与敏感领域的“风险距离”，将风险控制在安全阈值内触发后续动作，避免关键词匹配导致的误判。

## 2. 降级机制（Safety Fallback）

### 触发条件
任一分类器标记为“风险超过预设安全阈值”
- 不直接拒绝请求
- 不输出错误提示

### 平滑降级过程
静默路由到 Claude Opus 4.8 处理
- Claude Opus 4.8：耐低风险模型，安全稳定，擅长处理高风险请求

### 用户体验
- 无感知降级
- 体验不中断
- 无状态变更提示

### 分层安全过滤 + 动态计算资源调度
- **算力全用户**：保障高难请求安全处理，最大限度保证正常体验
- **高 Anthropic**：根据风险细分对高难任务 Fable 5 之外，满足大众需求的同时保持核心力安全

### 实际运行效果（官方数据）
- **>95%** 低风险请求直接由 Fable 5 处理
- **<5%** 高风险请求降级至 Opus 4.8 处理
- 对绝大多数用户完全无感

## 3. 安全路由的架构价值

### 行业对比：传统方案 vs Fable 5 方案

| | 传统方案 | Fable 5 方案 |
|---|---|---|
| | 发布两个独立模型版本 | 同一模型 + 动态安全路由 |
| A | 公众版（低安全，能力受限） | EZ Fable 5 同一底层模型 |
| B | 高性能内测版（高安全访问，仅高可信用户） | 路由层动态调度，匹配不同风险等同策略 |
| ✗ | 普通用户两个版本任务分支 | ✓ 能力与安全兼顾 |
| ✗ | 研发维护资源投入大 | ✓ 资源全局节省与聚风险 |
| ✗ | 无法自动适配，体验割裂 | ✓ 安全透明与核心计量隔离 |

### 架构价值
- 解决“AI 能力越强，潜在滥用风险越高”的行业矛盾
- 在不降低核心安全标准的前提下，提供高精度优业务能力
- 模型工程化领域的重要突破
- 支撑金融、法律、政务等高安全要求行业的快速落地

## 4. 实际落地的技术约束与“对齐税”

### 技术约束
- 多层过滤需隔离大安全风险，但误判为高风险，触发降级机制
- 静默降级到 Opus 4.8：性能高，速度更慢
- 用户难以察觉，或视为 Fable 5 能力不足

### “对齐税”（Alignment Tax）
为满足安全合规，牺牲部分正常场景的实际使用能力
- **安全保障** VS **业务适配性**

### 行业影响案例
- **医疗协同场景案例**：因安全策略触发屏障级，模型对专业术语的灵敏度降低约 11%，客户满意度下降显著
- **企业安全分析案例**：模型对内网攻击链路敏感度降低约 8% 造成漏报增加，导致安全团队响应延迟显著

### 误判概率（第三方监测）
**8% – 9%** 假阳性误判发生概率
- 人工智能分析（Artificial Analysis）

### 约束结论
对抗滥用风险越高模型的间用户，安全能力与业务体验需在之间仍存在显著的边界条件。
