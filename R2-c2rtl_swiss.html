02
进门体检 + 三硬条件 沙子进不去 · 粒度可控 · 反馈可追
G1
导入 Golden
确认 C 参考模型能跑出结果。C 是锚，不是起点。
G2
动态体检 UBSan
溢出 / 越界 / 未初始化 / 除零。触发即阻断。
G3
静态体检 CBMC
全路径推 UB。不跑 C 也能拦住带病 Golden。
①干净 C 锚
有 C 才能子模块独立验证与纠错。无 C 只靠自然语言规格 → 直接崩溃。
②单函数粒度
编译器拆分 · 拓扑序生成。拆到输入空间可穷举、错点可定位。
③精确反馈
定位器 + 压缩器 + EDA 桥接。证据给 LLM，不是「错了重做」。
Sub-agent：每轮独立执行——只传产物（代码 + 错题报告），不传思考噪声。隔离为质量，不为并行。
03
三条路径 · 同一验证框架 HLS 是可选插件，不是架构主体
PATH A · 主路
LLM 直写 RTL
数据通路 / 位操作 / 小 FSM / 精度敏感。商用 EDA 验证 + 错题闭环修复。拔掉 HLS 仍完整运转。
PATH B
HLS 兼容直编
pragma 齐 · 循环有界 · 接口规范 → Catapult / Vivado HLS。确定性翻译，不靠 LLM。
PATH C
重构 + HLS 调优
LLM 先改写成 HLS 友好 C → 综合 → 读报告调 PPA。深 pipeline / 资源共享场景。
04
经验飞轮三层 冷启动 → 温启动 → 热启动 · 复利不靠更大模型
L1
已验证模块复用
叶子函数跨设计重叠高。验过的 clz / pack 直接挂接，不重复生成。
L2
错题模式注入
subnormal / 对阶进位等病例写入 Prompt。首次质量靠经验，不靠撞运气。
L3
调试策略检索
同类 mismatch 先查历史修复路径。缩短收敛轮数。
05
三约束 · 白盒铁律 · AutoResearch 切位

三约束 · 敢用的笼子

  • AST 分解确定性——怎么拆不靠 LLM
  • 验证绝对——穷举 / SAT 完备，超时降级兜底
  • 错题可追溯——输入 · 期望 · 实际 · 行号可审计

铁律：白盒 RTL——变量名保留、结构保留、人可逐行对照 C。黑盒即使验证过也不敢用。

AutoResearch 切位

  • 切上层：Prompt / 错题压缩 / 经验检索
  • 不切底层：EDA 端到端（太慢 · license）
  • 指标：首次通过率 · 收敛轮数

切对是金矿，切错是无底洞。经验池 = 记忆飞轮在 C2RTL 的实例。
