# 4 · 5 执行引擎 判定架构

R2-Vplan 五层方法论 · 第四五层

让量化体系变成可预测的自动化行动。第四层管「怎么验对跑起来」，第五层管「怎么判定不用开会」— e=r-y，绿放行红退回。

## 第四层 · 执行引擎 — 翻译：格式转换。同一份 Feature 矩阵，PSS / DOM 在此层分叉

### DoD 交付物定义完成
用动作不用百分比。正确：「上板启动 MCU 并读取 IO 口数值」而非「Agent coding 80% done」

### Work List 只列必须做的事
四方面：TB infra · Coverage · Checks · Stimulus。不列有趣但非必须的事

### CI CI 驱动回归
自动编译–仿真–回归，每次 RTL 更新自动触发。把 agent 的任务从「判断」降级为「查证」，是这个架构能用在 CI gate 里的前提

## 第五层 · 判定架构 — Oracle 五层机械签核流水线。L2 – L4 机械比较器，L1 / L5 人校准

| 层 | 机制 | 判定者 |
|---|---|---|
| **L1 意图声明** | VPlan 通过结构化评审了吗？使命声明完整？Feature 矩阵覆盖所有需求？通过 → 流水线激活 | 人设定 r |
| **L2 可执行契约** | 期望 vs 实际比对。err=0 放行，≠0 退回。不关心数据从 formal proof 来还是 simulation coverage 来 — 只看 diff。 | 机械比较器 |
| **L3 验证架构** | Formal 报告 + 仿真回归日志 + 通过率 + 覆盖率 → 四个数字全绿放行。架构合规 Formal，复杂动态交仿真。 | 机械比较器 |
| **L4 质量内建** | TDD 三层全过。回归失败率稳定。瑞士奶酪：每层都有孔，叠起来穿透概率归零。 | 机械比较器 |
| **L5 工程效能** | 连续绿灯多少天？VPlan 验收通过率？效能指标持续达标后 — 签一个字。 | 人签字 |

## L2 可执行契约 · 在四个验证维度上的不同形态

| 维度 | L2 形态 | 示例 |
|---|---|---|
| **Formal** | property → proved / CEX | SVA 断言穷举证明：CSR 读写权限、指令编码合法性。工具自动返回 proved 或反例。 |
| **仿真** | Step-and-Compare | 同一测试程序同时在 RTL 和参考模型上跑，每条指令退休时自动比对 PC + 寄存器 + 内存写三路 diff。 |
| **Emu / FPGA** | C 断言 + 性能基线 | C 测试中的断言校验内存一致性 + IO 口数值。性能计数器作体检 — IPC 偏离基线即使用例全 PASS 也要报警。 |
| **QEMU** | 固件行为 vs Golden | 同一份 C 固件在 QEMU 和参考模型上跑，比对 printf 输出 + 外设响应序列。不碰时序，只验逻辑。 |
