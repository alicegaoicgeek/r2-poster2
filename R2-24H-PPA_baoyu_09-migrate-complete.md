# R2-第一圈到第三圈 · 方法怎么迁走 · 完成判据

09/09 · R2 芯片智能体

诊断越写越具体；三路真不同；失败可分类；基线仍固定。

## 前几圈把 loop 卫生做干净

- 统一口径：版本/脚本/环境/约束/场景全记录
- 三路诊断：结构 / 工艺 / 布局三路独立取样
- 失败即入库：原因归类、证据留存、可复现
- 计数换挡：无改善达上限，规则切换不纠缠
- 固定基线：同一把尺子量到底，不随意改口径
- 人签停表：达到完成判据，由人签字停表

## 迁移管法（不迁移某次 WNS 故事）

01 固定原点：基线不动、口径不变、尺子不换
02 先工具后分数：工具链/脚本/报表先固化
03 三路取样：结构 / 工艺 / 布局三路真不同
04 失败入库：原因可分类、证据可复现
05 计数换挡：达上限即换挡，防止空转
06 人签停表：满足完成判据，人工签字停表

## 完成判据（缺一不可）

01 合法停表：达到停表换挡条件
02 门禁通过：三路门禁全部通过
03 INDEX 自 v0 连续可审：从 v0 起，版本连续、可审计
04 相对固定基线对比：长期用同一基线做相对对比
05 人已最终签核：负责人最终签字确认

## GOAL（六项）

- 等价优先：功能等价是前提
- 锁定 WNS / TNS / Area：三大指标同时看齐
- score 公式：可复算、可解释、可审计
- 版本上限：每圈单一路径上限明确
- 门禁路径：三路门禁独立通过
- 停表换挡条件：无改善达到上限即换挡

## 人签核要求

- 抽查 FAIL 原因：从入库样本中抽查原因分类与证据链
- 可复算 score：随机抽样复算 score，与系统结果一致才签

## Dark 承诺

loop 转起来 PPA 才开始；
loop 管起来 PPA 才结束得了。

## 四步流程

### STEP01 锁版本与基线
- 固定基线与口径
- 锁版本 / 脚本 / 环境
- 记录约束 / 场景
- 生成基线报表（v0）

### STEP02 诊断三路门禁计分记账
- 三路取样 / 三路门禁
- 计算 score 并判定
- 失败入库分类
- 计分 / 记账 / 留痕

### STEP03 无改善换挡
- 达到版本上限
- 无改善（score 无提升）
- 触发换挡规则
- 进入下一条路径

### STEP04 停表人签
- 满足完成判据
- 负责人审查与签字
- 合法停表
- 归档与对外发布
