# R2-24h 交付一个工业级IP：跑通 PPA 优化

R2-24H PPA · INDUSTRIAL IP · 01 / 09

## 01 PREREQUISITES

签出在前，PPA loop在后

- 四千行 RTL（≥ 4K 行）
- 一万六触发器（≥ 16K FF）
- 深水区 IP（≥ 中等复杂度）
- 缺基线，禁启（无签核基线不启动）

## 02 LOOP STRUCTURE

01–07 七步，循环迭代

01 读 INDEX（基线/历史/禁忌）
02 诊断（定位 PPA 瓶颈）
03 三三制（3×3 候选方案生成）
04 门禁（合规/资源/风险检查）
05 score（量化打分与排序）
06 写回（结果入库与 INDEX 更新）
07 换挡/停表（继续/回退/停车）

## 03 SCORING（打分公式）

| 指标 | 权重（系数） | 说明 |
|---|---|---|
| 面积 (Area) | 0.50 | 越小越好 |
| 性能 (Perf) | 0.35 | 越快越好 |
| 功耗 (Power) | 0.15 | 越低越好 |

面积恶化 > 10% → 惩罚系数 0.5

综合得分 = 0.5×面积 + 0.35×性能 + 0.15×功耗（归一化到 0–100）

## 06 DARK FLOW（自动闭环流程）

读 INDEX → 诊断 → 三三制 → 门禁 → score → 写回 → 换挡/停表

- 读 INDEX：基线/历史/禁忌 · 最优/模板/约束
- 诊断：定位瓶颈 · 量化问题
- 三三制：四大策略 × 三档强度 · 生成 9 个候选
- 门禁：合规/资源/风险 · 不通过即淘汰
- score：公式打分 · 排序与决策
- 写回：结果入库 · 更新 INDEX
- 换挡/停表：继续/换挡/回退/停车 · 人签停表

## 07 RULES（七条铁律）

1. 基线不动：基线只读，所有优化基于副本与变体。
2. 失败入库：失败方案与原因必须入库，形成禁忌库。
3. 无改善换挡：连续两回合无有效改善，必须换挡。
4. 人签停表：达到完备条件需人签字，方可停表交付。
5. 尺固定回数：默认最多 40 回合，可由人调整上限。
6. 数据可信：所有数据可复现，可追溯，可对比。
7. 安全合规：不越权，不绕过流程，不牺牲正确性。

## 08 COMPLETION（完备条件：五条同时满足）

- 相对基线：面积 ≤ +10%，性能 ≥ 目标，功耗 ≤ 目标
- 连续两回合无可行改善，或已达预设回合上限
- 通过门禁与签核（功能/时序/功耗/DRC/LVS）
- 结果与证据（脚本/日志/报告）完整入库
- 人签字确认，状态为「停车」，形成正式交付包
