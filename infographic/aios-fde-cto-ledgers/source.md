|  三次用旧范式打新范式                                                                            |
|                                                                                                  |
|  误区 1  换一个更强的模型就好了                                                                  |
|          真相：模型既当选手又当裁判；没有 Oracle，能力越强，错误传播越快。                        |
|                                                                                                  |
|  误区 2  让 AI 自由发挥，提效最快                                                                |
|          真相：没有保护原语时，人就是保护原语；工程师更不敢放手。                                |
|                                                                                                  |
|  误区 3  加厚审批流程就能管住 AI                                                                |
|          真相：旧流程叠加 AI 只会增加等待；需要重新划分 AI、EDA 与人的责任。                      |
