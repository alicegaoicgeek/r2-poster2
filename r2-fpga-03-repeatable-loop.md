# R2-FPGA

## 方法 3 · 建立可重复的优化 Loop

将单次优化经验转化为可重复的 Loop：

| # | 步骤 | 说明 |
|---|------|------|
| 1. | 定义目标边界 | 明确频率、面积、目标器件、工具版本 |
| 2. | 跑公平基线 | 原封不动先综合一轮，记录 WNS、TNS、资源占用 |
| 3. | 判断差距 | 对比目标与基线，量化优化空间 |
| 4. | 解释差距 | 使用两步法（全局表 + 模块级定位）分析根因 |
| 5. | 证据驱动修改 | 先尝试约束和布局策略，再考虑 RTL 修改 |
| 6. | 相同条件重验证 | 修改后必须在相同测试条件下重新运行功能仿真 |
| 7. | 持久化记录 | 每轮记录时序结果、资源变化、关键路径、失败尝试、环境信息 |
| 8. | 迭代或停止 | 达到目标、证明不可行、或触及资源上限时停止 |

## Loop 规则

- 改代码前固定测试向量，禁止事后挑选简单用例
- 根据瓶颈类型匹配策略：BRAM 延迟优先插流水线，组合逻辑优先等价改写，扇出问题优先手动复制，布局问题优先调整策略
- 所有实验记录写入独立文件，禁止仅保存在 AI 对话窗口中
