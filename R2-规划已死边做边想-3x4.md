# R2-规划已死，边做边想

AI 时代认知顺序已经反转：不是先想清楚再做，而是边做边想才能搞明白

## [一] 旧前提：先想清楚，再动手

软件行业几十年的工作流——调研、规划、评审、动手——
都建立在一个隐含前提上：构建成本很高，必须先想清楚再动手。
PM 的 PRD 评审、架构师的技术方案评审、CTO 的季度 OKR，
本质上都是这个前提的产物。

发散 >>> 收敛 >>> 发散 >>> 收敛 >>> 交付

**这个前提塌了**

一个工程师同时跑 7 个 Claude Code，一周产出等于以前一个季度。
任何人提个想法，几分钟就能跑出可运行 demo。
模型输出是非确定性的——同样的输入可能产生不同结果。
构建不再昂贵，结果不再可预测，「先想清楚」在物理上已经不可能。

## [二] 新范式：用可运行的实践换可验证的认知

争论 API 方案怎么选？不找白板画半天——
直接生成三个可运行的 PR。
不是三个 PPT，不是三个草稿，
是三个能跑、能看到所有调用方实际影响的 PR。看完就不争了。
当构建成本足够低，争论就变成了最贵的事。
以前的认知靠事前推导和文档对齐，
现在靠可运行物和真实反馈。
「写代码前必须先写设计文档」的仪式也该砍掉。

## [三] 组织节奏：做一点发一点，及时规划

改成 just-in-time planning：
不攒大版本，不预热太多，做一点就发一点，看反馈再优化。
让真实用户反馈替代一帮人坐会议室里猜。
你得每周都能发布新功能。

做 >>> 发 >>> 真实反馈 >>> 再做

瓶颈从编码挪到了 Oracle 和 Loop。

## [四] 人还在：判断变便宜，责任不会

代码越来越便宜，判断越来越贵——
决定做哪个方向、哪种切法、做到什么程度收手，模型帮不了你。
AI 能出主意，没法替你吵架，更没法替你签核。
方向也没消失，只是变短了：
愿景就是 3~6 个月，形式是一个能跑的原型。
AI 时代不是不需要思考，
而是思考不能再全部堆在动手之前。

R2 芯片智能体
