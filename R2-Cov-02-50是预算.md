# 50 是预算，不是名单

R2-Cov · 回归筛选 · 02 / 06

> "大多数人听到'把 1000 个测试砍到 50 个'，第一反应是恐慌——那 950 个不测了？但这个问题的前提是——**50 个是固定的**。如果 50 个每次重算呢？今天的 Top-50 和昨天完全不同。覆盖总量在一个月内接近 1000 个。只是每次只跑最相关的 50 个。**预算不变，内容随情况变。**"

## 方法 · 每次重算

### 1 打分
CI 对 1000 个测试逐个打分，覆盖率关联 + 历史失败 + 模块关联

### 2 筛选
取分数最高的 50 个进入仿真，不固定名单

### 3 回写
仿真 pass / fail 结果回写，喂给模型

### 4 重训
每周重训，下次打分更准。持续进化

ARM 和 Siemens 独立验证了这套内核，两套算法同一个函数签名：变更 → 排序 → 子集 → 执行 → 反馈 → 进化

## 团队对比

### 团队 A · 静态名单
手工挑 50 个，三月不变。新模块加入 — 零覆盖。low power bug 漏到硅后，返工成本 200 万。

### 团队 B · 动态预算
每次 CI 自动打分 Top-50。新模块自然浮出。第三次 CI 抓到 bug。50 不是固定名单，是每次重算的预算上限。

---

**50 是预算，不是名单。名单是静态的，预算是动态的。预算是系统进化的前提。**

不是因为预算宽松了产出才增加，而是因为预算不变、内容重算——窄约束逼出了筛选，筛选净化了输入。
