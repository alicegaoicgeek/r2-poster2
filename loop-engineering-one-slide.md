# R2-循环工程 · 从写提示词到设计循环系统

@KK雷 · 一起成为AI未来领袖

## CHAPTER · 01 别再手动指挥 AI 了 [ 2026 新范式 ]

循环工程（Loop Engineering）：从「一句句写提示词」到「设计一个自己会跑的系统」—— Google 工程师 Addy Osmani 正式命名的高阶 AI 协作方式。

```
规划 PLAN → 执行 EXECUTE
    ↑              ↓
迭代 REFINE ← 检查 CHECK
         [ LOOP ]
```

系统自主转圈，人类只设目标和验收  
未来属于会「设计系统」的人，而不是只会写提示词的人。

## CHAPTER · 02 先认识这 5 个词 [ 术语速查 ]

|---|---|
| 循环工程 | 设计自动运行的系统，替你去指挥 AI。 |

五个词记住了，循环工程就懂了一半。

## CHAPTER · 04 一个完整指挥 = 5 大组件 + 记忆 [ 系统解构 ]

| | 组件 | 说明 |
|---|---|---|
| 01 | 自动化调度 Automation | 定时触发：每天扫描新 bug、CI 失败 |
| 02 | 工作树隔离 Worktree | 多个 AI 并行干活，文件互不打架 |
| 03 | 技能文件 SKILL.md | 规范、知识、历史决策写死，启动即读 |
| 04 | 连接器 Connectors | 让 AI 真正操作 issue、数据库、Slack |
| 05 | 子代理分工 Sub-agents | 一个写、一个审：制衡分离，避免自评 |
| + | 外部记忆 External Memory | Markdown 记录已完成与待办，形成闭环 |

像控制系统一样：有正负反馈，人类可随时介入纠偏。

## CHAPTER · 05 提示词 vs 循环 差在哪？ [ 正面对比 ]

| | 提示词工程 | 循环工程 |
|---|---|---|
| 驱动 | 人类手动，每步监督 | 系统自主：发现→执行→检查→迭代 |
| 记忆 | 靠上下文窗口，易丢失 | 外部持久化，跨会话连续 |
| 角色 | 键盘手，全程在场 | 系统设计师，最终把关 |
| 场景 | 简单、单次任务 | 复杂、长期、多步骤 |

比喻：提示词 = 开手动挡；循环 = 设计自动驾驶，但验收仍归你。  
不是取代，是分层：简单任务用提示词，复杂持久任务用循环。

## CHAPTER · 06 四级跳：它从哪里来？ [ 演进路径 ]

这是 AI 使用方式的自然演进，不是凭空造词。反思循环可将中等模型逻辑执行能力提升 30% 以上。

| 层级 | 名称 | 说明 |
|---|---|---|
| LV.1 | 提示词工程 Prompt | 写好单条指令 |
| LV.2 | 上下文工程 Context | 构建丰富动态上下文 |
| LV.3 | 运行环境工程 Harness | 打造智能体运行框架 |
| LV.4 | 循环工程 Loop | 框架自主定时运行、自我喂养、迭代 << 当前层级 |

一人设计系统，多个智能体 7×24 并行作战——这就是杠杆。

## CHAPTER · 07 爽归爽，四个坑要避

1. **验证责任永远在人** — 无人值守的循环，也会无人值守地犯错。只交付你亲自确认可用的成果。
2. **理解力负债 Comprehension Debt** — 不读循环的产出，你对项目的理解会快速变浅。
3. **认知投降 Cognitive Surrender** — 循环跑得最顺的时候最危险——别彻底放弃思考。
4. **成本失控** — 子代理多轮迭代消耗剧烈，必须设预算和停止条件。

把「人类判断环节」设计进循环，而不是按下启动就走开。

## CHAPTER · 08 把自己从「键盘手」升级为「系统设计师」

从今天起，用 AI 帮你设计第一个小循环——比如一个每日自动总结任务。先写扎实验收标准和技能文件，从小任务开始。

STEP 1 定目标 与 验收标准 → STEP 2 搭组件 从小循环起步 → STEP 3 人在环中 最终把关

系统可以替你跑，但最终价值，取决于你的理解和把关。
