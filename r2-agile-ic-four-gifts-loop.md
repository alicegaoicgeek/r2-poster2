# R2-敏捷IC四礼物: 硬件版 Loop Engineering

同一个第一性原理：  
反馈越快，系统越真；契约越硬，组织越会进化。

敏捷IC把软件敏捷翻译到硬件，并多出两条工程硬约束：

## 左侧六项（反馈带宽）

01 人与协作 → 反馈带宽  
02 可工作原型 → 唯一真相源  
03 跨职能团队 → 跨边界信息流  
04 响应变化 → 学习速度  
05 设计自动化与芯片生成机制 → 工具链即方法论  
06 尽早集成、基于反馈逐步完善 → 集成本身是发现机制

## 右侧四礼物

### 01 礼物一 | 重新切分组合边界
单点优化 → 价值流动  
单次 prompt → 持续收敛的迭代流

### 02 礼物二 | 让接口从碰撞中涌现
接口不是规划出来的，是失败模式撞出来的  
同错两次，写入 guardrails.md

### 03 礼物三 | 机器可执行的契约
Exit when 与 Between iterations run 必须是 CLI  
`npm test` / `gh pr checks` / `npm run build`  
**不靠 AI 自己判断**

### 04 礼物四 | 学习型组织持续进化
短迭代让成功可固化、失败可复盘  
Memory + Skills + Harness：人工修正回流系统，经验留在磁盘

## 底座

Memory · Skills · Harness

R2芯片智能体
