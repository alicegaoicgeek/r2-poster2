# 都江堰效应

R2-Cov · 回归筛选 · 04 / 06

2200 年前，李冰在岷江上修了都江堰。三道结构，没有闸门，没有电力，持续运转至今。它的工程逻辑，恰好是 R2-Cov 的架构逻辑。

## 三道结构 · 物理课

### 鱼嘴 切分
不抽水，不推水。只借弯道离心力——清水密度小走凹岸，浑水密度大走凸岸。水自己分开。

### 飞沙堰 排沙
一道低矮的溢流堰，高出河底仅 2 米。水从旁边流过，泥沙被旋流横向排出。它不挡水，它坐水。

### 宝瓶口 约束
宽仅 20 米。不管上游来多少水，进成都平原的上限是 700m³/s。

宝瓶口窄 → 上游壅水 → 壅水遇到离堆 → 产生旋流 → 旋流就是飞沙堰。  
如果宝瓶口是敞开的宽渠，水全冲过去，没有壅水，没有旋流，飞沙堰不过是一堆泡在水里的石头。

## 翻译到验证

| 都江堰 | R2-Cov 验证系统 |
|---|---|
| 鱼嘴 | 自动从 spec 生成 covergroup，无需手写 coverpoint |
| 宝瓶口 — 20 米窄口 | 50 个测试的预算上限 |
| 壅水 — 竞争入选 | 只有 50 个名额，测试必须竞争入选 |
| 飞沙堰 — 旋流排沙 | 每轮 CI 自动淘汰低价值测试，保留命中率最高的 50 条 |
| 无闸门 / 无电力 | 全自动闭环，每次 CI 自动重算 |
| 水变干净 | 回归集命中率持续上升，遗漏 bug 数持续下降 |

## 500 vs 50

**500**  
失败淹没在 490 个通过的噪音里，排序模型无法区分相关失败和随机波动，回归集不会自我优化。

**50**  
每一次失败都是信号。50 条硬上限迫使每轮 CI 丢弃最弱的测试，替换为新的候选。洪水越大，旋流越强，排沙越狠。

**50 就是宝瓶口。窄创造壅水，壅水产生旋流，旋流驱动飞沙堰。窄是动力源。**

命中率提升不是因为测试数量多了，而是因为每条保留的测试都经过多轮筛选验证。宝瓶口不扩，通过量变大 — 不是口子大了，是水变干净了。
