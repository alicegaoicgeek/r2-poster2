# R2–等价抽象 × 不适用 × 带走物

## 等价抽象：硬件还没有时，三刀如何先跑起来

接口契约先行；需要被工具消费时，再投影为 IP-XACT 或 ip-exact。五层不是装饰名单，而是随成熟度切换的工作台。

| 层 | 是什么 | 主要服务哪一刀 | 切入 / 退出 |
|----|--------|----------------|-------------|
| Golden Model | 功能与时序意图的可执行参考（Python/C） | 第二刀 Oracle；反哺第一刀 | Sprint 0 就建；RTL 稳定后仍保留 |
| VIP | 总线/协议验证 IP，检查接口合规 | 第一刀 Stub 侧与第三刀对齐前 | 接口协议选定后接入 |
| Stub | 缝上的占位实现，契约在、行为可简化 | 第一刀解耦横切；支撑并行 | 边界未定前即可立；对端就绪后替换 |
| TLM VP | 事务级虚拟平台，跑软件可见流程 | ④⑤软件栈早启动 | 软件栈要并行时切入 |
| FPGA | 接近实现的原型，补时序与真实外设 | 第三刀端到端加压 | R2/R3 按需；不替代早期 Golden/Stub |

**切换条件**：需要分钟级、可重复的对错裁决时，优先 Golden Model，不要跳去 FPGA 做入肉波形。需要多人并行、对端未就绪时，用 Stub；需要跑驱动与中间件时，上 TLM VP；需要闭合真实外设与时序风险时，再上 FPGA。

## 不适用：不必把三刀当成万能刀谱

| 场景 | 建议 |
|------|------|
| ① 纯 Hard IP 挂接、边界已经验证 | 第一刀轻量确认契约；第二刀可薄；重点在第三刀 ICD 冻结 |
| 单 IP / 单模块、无多特性拼合 | 可只做第二刀短闭环；发布火车可省略 |
| 无清晰数据包 DITL、偏控制面 | 改用事务路径或配置路径定义缝隙 |
| ⑥ 成熟子系统黑盒交付 | 三刀落在黑盒边界外；盒内不必再切 |
| 探索性预研、尚无 Oracle | 先建 Layer 1 参考答案，再谈分钟级 |

## 带走物

- **01** F = G × H × I，乘积而非加和；每一刀砍一个因子
- **02** 顺序：隔离 → 闭环 → 拼合，因果锁死，不可颠倒
- **03** 切法：沿 DITL，不切模块边界
- **04** 角色：Peter 定战场，Andy 切前两刀，Vicky 建裁判，Fred 排火车
- **05** 裁判：机械 Oracle，不要人肉 review 充当唯一门槛
- **06** 接口：机器可读 ICD（Pipe/Valve/Translator），r2-specx 单一事实源
- **07** 路径：六种集成方式都要 ICD，但来源不同；三刀承重按①–⑥分流
- **08** 发布：R1 → R2 → R3 纵向切片；端到端从第三刀开始
