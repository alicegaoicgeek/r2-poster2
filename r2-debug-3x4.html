SIX METHODS · 六法（决策层）
①并案
分桶后选共享信号最少、信息增益最大的先修
②比差
先看两个版本变了的模块/信号，不看全量
③以动制静
卡住就动一下，画面差异即线索
④回滚
攻击 bug 的生存条件而非本体（git bisect）
⑤人类直觉
从海量数据降维到 3–5 个 failure signature
⑥经验回流
每次结果校准下一步方向，负反馈加速
EIGHT STAGES · 八段闭环（工厂层 · 出错回流预防）
阶段子 skill职责
① 预防 r2-xpattern 反模式回灌 / 规则 / Linter
② 检测 r2-xfail + xindex.stats 编译 / 仿真 log → 结构化 FAIL 事件
③ 分诊 r2-triage N 个 FAIL 聚类 / 优先级 / 归属（N>1 才启用）
④ 定位 r2-xindex.resolve + xwave loc_id / @time → 源码锚点 + 时间窗
⑤ 根因 r2-xtrace ⇄ xwave 观察 → 假设 → 验证 → 精化

⑦ 签核 r2-xsign 重跑 + 对比 specx 契约，确认不回归
