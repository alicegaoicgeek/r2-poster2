# R2+ClaudeCode 智能体架构

AI Agent = 大脑（思考）+ 记忆（经验）+ 工具（行动）+ 循环（执行）+ 护栏（安全）

## CLAUDE CODE 记忆系统（Memory）
存储层 + 维护层 + 行动层 = 离线运转伙伴

### 1. 存储层（Memory Files / 海马体）
- 结构化 MD 文件
- MEMORY.md 索引
- <=200行 / <=25KB
- 用户可控可删
- MEMORY.md

### 2. 维护层（Dreams / REM 睡眠）
- 受 REM 睡眠启发，后台异步整合
- 触发：>=5 次对话 且 >=24h
- 手动 /dream

### 3. 行动层（Conway / 清醒执行）
- 7x24 常驻 Agent
- 事件驱动唤醒 / Webhook 触发
- nightly regression / CI 触发
- 任务完成后睡眠

## 用户输入 / 目标
「帮我定位这次 SoC regression 的覆盖率空洞」

### 目标拆解
- 理解 spec/约束
- 分析失败 log
- 制定补洞计划

**Agent 循环**（持续迭代直到签核通过）

## 工具（Tools）
让 Agent 与外部世界交互

Grep · Read · Bash · Sim · Cov · Wave · 更多

### 工具调用流程
{name, args} -> dispatch（分发）-> exec（执行） ↔ result（结果）

## 决策循环

- **① 感知（Perceive）**：读 RTL / RAL / spec · 理解当前验证状态
- **观察（Observe）**：读 sim log / cov 报告 · 感知覆盖率变化
- **大脑（LLM）**：推理 + 规划 + 反思 · [Plan] [Review]
- **② 思考（Think）**：定位 root cause · 决定补哪条 stimulus · 规划下一步验证动作
- **③ 行动（Act）**：改 sequence / SVA · 跑 smoke / regr

检索/注入上下文 · 调用工具

## 决策机制
决策 = 当前感知 × 已存储记忆  
- 决策不是独立模块，是记忆作用于感知的输出  
- 记忆质量决定决策质量  

### 协作模式（7种常见模式）
1. Workflow 固定流程
2. ReAct 边想边做
3. Plan & Execute 先规划执行
4. Tool Use 调用外部能力
5. Function Calling 参数化
6. Reflection 做完再复查
7. Multi-Agent 多个AI分工

停止条件：签核通过 / 达到最大迭代 / 超出预算

## 安全护栏
所有行动必须经过安全检查，确保可靠、可控、可追溯

待执行动作 → 验证 → 权限范围 → 预算控制 → 允许执行  
（任一失败则拒绝）

## 智能体生命周期
目标设定 → 感知理解 → 思考规划 → 行动执行 → 反馈学习 → 进化提升（循环迭代）

## 组件说明
- **记忆模块**：短期记忆/长期记忆、向量记忆/知识图谱
- **规划模块**：任务分解、步骤规划、策略生成
- **推理模块**：逻辑推理、因果推理、反思优化
- **决策模块**：目标对齐、方案评估、行动选择
- **领域知识**：企业知识、FAQ、文档/手册
- **提示工程**：提示模板、上下文管理、记忆注入、指令优化
- **工具与能力**：API调用、数据库、搜索引擎、代码执行器、文件操作、第三方服务、硬件设备
- **行动层（手和脚）**：行动执行器：接收指令 → 调用工具 → 执行任务 → 返回结果

## 优秀智能体的特征
- ✓ 目标明确，有主动性
- ✓ 感知全面，理解准确
- ✓ 推理合理，决策可靠
- ✓ 行动高效，结果可控
- ✓ 持续学习，不断进化
- ✓ 记忆持久，经验复用
- ✓ 工具熟练，交互流畅
- ✓ 安全合规，风险可控
- ✓ 自主循环，闭环执行

---

**一句话总结：** AI Agent 的本质是一个「能感知、会思考、能行动、可迭代、有记忆、受约束」的智能系统。  
它不是一次性回答，而是一个持续循环、不断自我修正、最终完成验证签核目标的过程。
