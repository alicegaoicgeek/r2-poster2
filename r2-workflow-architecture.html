R2-Workflow 产品架构图

Evidence-gated Agent execution — claim completion only with proof
定位 消极验证框架：不承诺 AI 一定做对，只保证证据不足时不允许声称完成。约束 Agent 行为顺序与状态机，不修改模型。
价值 把「我完成了」变成可验证命题——与 TDD / Code Review / CI 同构：不合格声明 → 机器拦截。

2

三条主流程（状态机）

🎯

目标驱动

  • 拆 goals → 逐目标执行
  • Gn 有证据 → 才开 Gn+1
  • 多步骤实现任务
🔧

调试诊断

  • 复现 → 多假设并行
  • 证伪测试 → 验证修复
  • 报错 / 异常场景
🔍

审查驱动

  • 记录 findings
  • 逐项解决 → Gate 检查
  • 代码审查 / 质量巡检
⚡

快速模式

单步可瞬间完成 → 执行 + 单次验证，简化 Gate

💬

轻量模式

纯问答 / 咨询 → 跳过重型流程，保留审计边界

3

📝

发现清单

📎

证据日志

命令·输出·时间戳·追加写

⚙️

编排入口

SKILL.md

工作流编排·能力边界声明

4

验证门（Gate）— 核心产品承诺

检查项拦截条件
任务分类模式与场景不匹配 → BLOCK
逐步证据goal 缺 ledger 记录 → BLOCK
自动验证test / lint / typecheck 未跑 → BLOCK
发现关闭findings 存在 open → BLOCK
🛡️

失败时输出

不声称完成，只报告：改了什么 · 验证了什么 · 剩余风险

5

产品边界（拍板口径）

我们交付

  • 文件驱动状态机，会话中断可恢复
  • 纯函数 Gate，机器可判 pass/block
  • 跨平台移植（Skill / Rules / 自定义 Agent）
  • 与 R2-AIOS Harness 编排层咬合
VS

我们不承诺

  • 不修改模型权重或能力上限
  • 不保证「零幻觉」「一定做对」
  • 不靠口号灌输纪律，靠结构展示
  • 验证命令需按项目适配
6
