# R2-四渡赤水 × r2-debug

**一句话**
r2-debug 六法对应的是四渡赤水的四段式：
找到缝隙 → 观察变化 → 制造信息 → 直捣中枢。

**核心命题**
调试不是证明自己猜对了。
调试是通过每一步行动制造新信息，让下一步选择更准。

## 四段主链路

① 找到缝隙 → ② 观察变化 → ③ 制造信息 → ④ 直捣中枢

## ① 并案：分桶之后先打哪个

**四渡赤水动作**
选四渡烈，不只是黔军弱，而是王粲矛盾让薛岳不真救；贵州方向还连接云南，打穿黔军打开向西通道。

**调试映射**
先修 log 最清晰、耦合最松、修完能暴露其他根因线索的 failure cluster。

**r2-debug 能力边界**
并案把 300 个 failure 按 log 模式聚成 3–5 类。
能力边界：信息增益评估仍靠人判断。

## ② 比差：先看变了什么

**四渡赤水动作**
每到新位置，先侦察敌军部署变化。

**调试映射**
版本 N-1 → 版本 N，只看变了的模块和信号。
纪律：迭代速度必须快过认知惯性。
规则：一个实验超过 15 分钟还没结果，换更小的实验。

**r2-debug 能力边界**
比差定位模块、信号、配置的变化面。
能力边界：自动生成对比实验仍需增强。

## ③ 以动制静：制造变化，从变化里看新东西

**四渡赤水动作**
佯攻贵阳城，不为占领，而是让蒋介石确定反应。

**调试映射**
怀疑模块 A，不是找证据证明 A 有问题，而是构造只测 A 的最小 test case，让 bug 暴露它和 A 的关系。
规则一：实验覆盖面必须够宽。
规则二：假设必须有区分力。
规则三：做完实验后，剩余假设空间必须显著缩小。

**r2-debug 能力边界**
以动制静通过置信度排序和最小实验制造信息。
能力边界：实验可区分性仍需量化。

## ④ 回滚：攻击 bug 的生存条件

**四渡赤水动作**
威胁贵阳，不是真打贵阳；目的是让滇军离开金沙江。

**调试映射**
bug 存在需要条件同时满足，只要改变其中一个条件，bug 行为就会暴露。
四类条件：时间条件 / 空间条件 / 历史条件 / 配置条件。

**r2-debug 能力边界**
回滚定位哪个 commit 引入了 bug 生存条件。
数据锚点：分析时间 34 秒 vs 传统二分 1968 秒。
能力边界：跨 failure 条件共享分析仍可演进。

## ⑤ 人类直觉：从海量数据里看到结构

**四渡赤水动作**
不看 40 万敌军总数，而看川军、滇军、黔军、中央军各自保地盘的利益边界。

**调试映射**
几万行 regression log → 3–5 个 failure signature；几十个 commit diff → 模块级差异；几百个 test pass/fail → group 全灭或零星失败。

**r2-debug 能力边界**
工具做聚类，人做判断。
能力边界：降维维度选择仍靠经验。

## ⑥ 经验回流：每一步校准下一步

**四渡赤水动作**
一渡发现北面太硬，二渡利用错觉打回东面，三渡验证佯动有效，四渡等到金沙江方向敞开。

**调试映射**
猜测 → 实验 → 结果 → 修正猜测 → 下一轮实验。

**r2-debug 能力边界**
历史 regression 结果给 commit 打分，越像曾经引入 failure 的 commit 越优先验证。
能力边界：验证结果回流模型仍需加强。

## 总映射

| 动作 | 含义 | 对应方法 |
|---|---|---|
| 软柿子 / 找缝隙 | 分桶后选信息增益最大的先修 | ① 并案 |
| 比差 / 以动制静 | 先看变化，看不懂就做实验 | ② 比差 + ③ 以动制静 |
| 佯动 / 制造信息 | 做实验看变化，不脑补原因 | ③ 以动制静 |
| 直捣中枢 | 攻击 bug 的存在条件而非本体 | ④ 回滚 |
| 信息降维 | 从海量数据中看到结构 | ⑤ 人类直觉 |
| 反馈闭环 | 每次验证结果校准下一步方向 | ⑥ 经验回流 |
