# R2-FPGA

## 方法 1 · 建立团队级 AI Skill

将已验证的 AI 工作流，写成可复用 Skill。每个 Skill 固定回答五个问题：

1. **触发条件：** 什么情况下启用该 Skill
2. **输入定义：** 向 AI 提供哪些文件、命令或上下文
3. **执行步骤：** AI 必须按什么顺序执行，禁止跳步
4. **成功标准：** 输出什么结果或指标即视为完成
5. **停止边界：** 什么条件下必须停止，防止无限迭代

## 首批应建立的三类 Skill

### Skill A：时序问题排查
- **触发：** 综合或布线后出现时序违例
- **步骤：**
  1. 读取全局时序报告
  2. 按模块归类最差路径
  3. 识别瓶颈类型（BRAM 延迟、组合逻辑、扇出、布局）
  4. 输出修复优先级表
- **铁律：** 禁止 AI 直接修改 RTL，必须先尝试约束调整和布局策略

### Skill B：RTL 时序优化
- **触发：** 排查 Skill 证明必须修改 RTL
- **步骤：**
  1. 定位具体路径
  2. 检查等价改写机会
  3. 仅修改被路径指出的具体模块
  4. 同步更新测试平台预期
- **铁律：** 修改后必须完成功能仿真，否则禁止进入时序对比环节

### Skill C：平台迁移
- **触发：** 需要将设计迁移到新 FPGA 平台或工具链
- **步骤：**
  1. 新建独立目录
  2. 提取设计信息
  3. 提取目标板卡信息
  4. 生成约束文件
  5. 生成比特流
- **铁律：** 原工程目录只读，所有改动在新建目录完成

用可复用的 Skill 固化经验，用标准化流程放大团队能力，让 AI 成为工程交付的可靠加速器。
