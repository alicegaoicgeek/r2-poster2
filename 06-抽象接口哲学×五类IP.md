# 管道 · 阀门 · 翻译

R2-Vplan · 五层方法论 · 抽象接口哲学

ABSTRACTION · INTERFACE

复杂系统靠分层，层间靠接口。IP 类型只改变每层填什么，不改变框架本身。

## 01 三层接口哲学

| 管道 | 阀门 | 翻译 |
|------|------|------|
| **信息通道结构** | **信息边界控制** | **跨层透明承诺** |
| 信息在独立通道上走，互不阻塞。三条独立通道：命令、数据、响应。 | 什么能过界，什么被挡住。接口冻结后只有约定的信息允许跨层。 | 同一份意图，多套物理实现。上层只面对接口，不感知底层差异。 |

## 02 五类 IP 验证落点

| IP | 切分 | 管道 | 阀门 | 翻译 |
|----|------|------|------|------|
| CPU / XPU | Pipeline 阶段（取指 / 译码 / 执行 / 写回） | AMBA 总线 = 信息通道 | MMU 页表权限 + 中断使能 | C → 汇编 → 机器码，ISS Lockstep 比对 |
| 算法类 | 数据流 / 控制流分离，标准接口层 | Mem 进 Mem 出归一化 | 定点精度限制 = 合法数据范围 | 浮点 → 定点 → RTL，bit-true 对齐 |
| 接口与互联类 | 协议分层 + Master / Slave 拓扑 | TX/RX 双通道 + 多链路并发 | LTSSM 状态机 + QoS 仲裁 | VIP 翻译为事务层 + 性能模型闭环 |
| 控制系统 | 控制环路 / 功率级 / 采样器分离 | 反馈信号线 = 闭环信息通道 | PID 限幅 + 饱和保护 | Simulink → HDL Coder → RTL |
| 全芯片类 | 按子系统切分，接口契约冻结 | 子系统间数据流 = 跨层通道 | 子系统接口契约 = 跨边界控制 | 子系统验证 → 全芯片集成 |

## 03 参考基准

| IP | 参考形态 | 替代 / 补充策略 |
|----|----------|-----------------|
| CPU / XPU | ISS 指令集模拟器 | Lockstep 逐指令 + TraceLog + printf 自检 |
| 算法类 | C / MATLAB 浮点定点模型 | bit-true 标准接口，需定点化转换 |
| 接口与互联类 | VIP 参考模型 + Python 性能模型 | 协议合规靠 VIP + 性能靠 Python 模型 + 真卡互操作 |
| 控制系统 | Simulink 浮点 / 定点模型 | 频域 + 时域分析 + HDL Coder 生成参考 RTL |
| 全芯片类 | 子系统级参考模型 + Boot 固件 | 子系统复用验证 + 场景集成 + Boot/Software 驱动 |

**总纲** VPlan 是问卷，问卷不绑定工具。CPU 验指令流，算法验数据精度，接口与互联验协议合规与并发吞吐，控制系统验环路稳定性，全芯片验子系统集成 — **五类 IP，一套框架，五层方法论，双路径落地，四维验证出口。**
