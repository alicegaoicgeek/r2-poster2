# R2-上下文服务：统一什么 / 不统一什么

FPGA · PPA 性能优化 loop · 与 Claude Code 无关的那一层

## 可统一：打包协议（骨架）

### Context Service · 交通规则
只规定「每轮窗口长什么样」  
不规定「怎么把 PPA 做漂亮」

- **P0 钉死目标**  
  Fmax / 面积 / 功耗；硬约束 · 止损

- **P1 结构化度量**  
  JSON / 表；不靠聊感觉

- **P2 瓶颈摘要**  
  report 外置；进模只抽热点

- **P3 代码切片**  
  相关路径；禁止默认整仓

- **P4 历史上限**  
  试法→涨跌；防震荡 · 防爆窗

每轮组装 → ctx_pack  
模型 → 工具执行 → Gate → state on disk

## 难统一：PPA 方法 / 个人习惯

### Plugins · 馅

- **设计策略包**  
  简单模块 vs 复杂 SoC；热点怎么定义、按哪层切

- **工程师 Skill**  
  先时序还是先资源；禁区、偏好、风险偏好

- **工具链 Parser**  
  Vivado / Quartus / 自研；report → P1/P2 字段映射

## 真相源在盘上，不在聊天记录。
