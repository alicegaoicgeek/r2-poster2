# R2洞见：Skill 既是便利贴也是 IP，从便利贴跃迁到 IP 需要三线激活

## 变化压力
场景变化 · 工具链升级 · 模型基座变化 · 其他 Skill 变更

叠加 → 适配冲突 → Skill 输出质量退化

## 线一：结构约束
Harness → 退化检测 → 修复流程 / 版本回滚

协议层（接口契约）+ 编排层（调度组合）+ 运行层（生命周期）

## 线二：反馈回路 · 四件套技术管线

人的隐性判断 + 执行轨迹（Raw-Trajectory）

1. **SkillOpt** — 信度分配：把 Skill 当可训练外部状态
2. **Skill蒸馏** — 因果链重构：不是压操作记录，是压领域知识
3. **SkillEval** — 度量闭环：量化评估 + 动态上下文注入
4. **Skill进化** — 版本写回 / 退役 · 多条进化路径

重新验证 → 检测适配冲突（闭环）

## 线三：组织背书
DRI 维护 + 流程嵌入 + 标注回库 → Skill 资产属性：便利贴 ↔ IP

## 判据：
结构约束 + 反馈回路（四件套）+ 组织背书 → 三线俱在 Skill即IP；缺一即便利贴
