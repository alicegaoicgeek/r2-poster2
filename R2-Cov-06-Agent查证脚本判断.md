# Agent 查证 脚本判断

R2-Cov · 回归筛选

06 / 06

## 01 智能体 · 每周蒸馏
读回归摘要 → 分析模式 → 蒸馏规则 → 更新规则库

输出：可执行的规则库。不是分数，是规则。

## 02 Agent 蜂群 · 每次 CI 并行查证
300 agent 并行，每个查一个测试，分钟级收表

## 03 确定性脚本 · 毫秒级排序
规则库 + 查证表 → 算分 → 排序 → Top-50

同一输入永远产出同一输出，无 LLM 随机性。

### 查证维度

| 查证 | 名称 | 问题 |
|------|------|------|
| 查证 1 | 覆盖率关联 | 覆盖文件是否包含 diff 文件名？ |
| 查证 2 | 历史失败 | 最近 30 天是否有 fail 记录？ |
| 查证 3 | 模块关联 | knobs 与变更模块有无关联？ |
| 查证 4 | 变更影响 | diff 是否触发敏感路径？ |

覆盖率关联 +30 | 历史失败 +20 × 失败率 | PM 覆盖 无条件 100 | 连续 4 周全绿 → 排除

## 核心判断

把 agent 的任务从判断降级为查证，是这个架构能用在 CI gate 里的前提。

两条路径叠加：运行效率（Agent 筛选）× 搭建效率（标准接口）。确定性是 CI gate 的生命线。
