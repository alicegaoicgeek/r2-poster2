# R2-七个维度的同构映射

一个 Loop 类似敏捷软件开发中的一个 Sprint：
有节奏的持续交付。

## 中心控制环
执行 → 感知 → 比较 → 调整 → 再执行
（Exit condition 在底部）

## Scrum Sprint vs Loop

| # | Scrum Sprint | Loop |
|---|---|---|
| 01 心跳 | 固定时间盒（2周） | /loop 15m 或动态自调节 |
| 02 交付物 | Potentially Shippable Increment | 每轮离 exit condition 更近一步 |
| 03 检查点 | Sprint Review 看产出 | Between iterations run: npm test 看命令输出 |
| 04 回顾 | Retrospective 改进流程 | evaluator 的 reason 注入下一轮 |
| 05 Backlog | Product Backlog → Sprint Backlog → Goal → 每轮拆解出的具体 step | Goal → 每轮拆解出的具体 step |
| 06 Definition of Done | 团队约定的完成标准 | Exit condition |
| 07 Scrum Master | 移除障碍、保护节奏 | 自动调度、worktree 隔离、hook 门禁 |

## 结论：
每一个 Scrum 结构要素，都在 Loop 中有对应机制。

两者共享同一个控制论结构：
执行 → 感知 → 比较 → 调整 → 再执行。
