# R2- Agent八大主流架构

从ReAct到自主智能体 · 知识图谱 · 00/09

## 01 ReAct 推理与行动协同
想→做→看反馈循环

想 Thought 推理 → 做 Action 行动 → 看 Observation 观察 → 循环 反馈后继续

- 适用：通用任务/工具调用
- 优点：简单直观，上手快
- 缺点：长链推理易偏航
- 特点：逐步决策，实时反馈

## 02 Plan-and-Execute 规划与执行分离
先规划后执行可回滚

用户目标 → 规划 (Plan) 分解步骤/依赖/顺序 → 执行 (Execute) 逐步执行 → 回滚/重试/调整

- 适用：复杂任务/流程型任务
- 优点：结构清晰，可控性强
- 缺点：规划质量决定上限
- 特点：先规划后执行，可回滚

## 03 Multi-Agent 多智能体协作
协调器+专科Agent+共享知识库

用户/任务 → 协调器 分配/路由/聚合 → 专科Agent … → 共享知识库

- 适用：复杂领域/多能力协同
- 优点：并行协作，能力互补
- 缺点：通信成本/一致性
- 特点：分工协作，共享知识

## 04 Reflective Agent 反思型
生成→反思→改→最多3-5轮

生成 → 反思 评估/发现问题 → 改进 修改/优化 → 循环 最多3-5轮或收敛

- 适用：高质量回答/写作/推理
- 优点：质量可提升，有自纠错
- 缺点：消耗Token/时间较多
- 特点：自我反思，迭代优化

## 05 Tool-Augmented 工具增强
LLM维护工具列表自行调用

用户/任务 → LLM 决策 选择工具 → 工具列表 检索/选择 → 调用工具 执行 → 返回结果 整合输出

- 适用：需要外部工具的场景
- 优点：灵活扩展，即插即用
- 缺点：工具质量/安全风险
- 特点：LLM维护并调用工具

## 06 Memory-Augmented 记忆增强
会话/短期/长期/向量四级

会话记忆 当前会话上下文 ↔ 短期记忆 近期交互信息 ↔ 长期记忆 持久化知识 ↔ 向量记忆 向量检索/相似

- 适用：多轮对话/个性化场景
- 优点：上下文连贯，个性化强
- 缺点：存储/检索成本较高
- 特点：四级记忆，分层管理

## 07 RAG Agent 检索增强生成
查资料再回答可溯源

用户问题 → 检索 (Retrieve) 向量检索/关键词 → 知识库 文档/网页/结构化 → 生成 (Generate) 基于证据生成 → 回答 (Answer) 可溯源输出；引用来源/证据

- 适用：问答/知识密集型任务
- 优点：准确性高，可溯源
- 缺点：依赖知识库质量
- 特点：检索增强，证据驱动

## 08 Autonomous Loop 自主循环
目标分解执行评估追踪闭环

目标设定 设定目标 → 分解计划 任务分解 → 执行行动 调用/执行 → 评估反馈 评估/打分 → 追踪学习 记录/学习

- 适用：长期任务/自主运行场景
- 优点：端到端闭环，自我驱动
- 缺点：复杂度高，调试困难
- 特点：自主循环，持续优化
