# R2-Loop 模板

## 1 Manual 型（手动触发，最常用）

启动「[Loop名称]」循环。

- 目标：[一句话定义完成状态，必须可验证]
- 每轮之间执行：[可机器执行的CLI命令，例如 npm test]
- 退出条件：[命令输出标准，绑定到退出码]
- 最大轮次：[3-20，中位数8]
- 第一步：[明确的第一步操作，防止冷启动]

护栏：
- 不得修改检查命令或退出标准来强制通过
- 不得跳过、禁用或绕过检查来满足退出条件
- 如果多轮后卡住，停止并报告阻塞项，而不是玩弄指标

[涉及测试时追加：
- 不得削弱、删除或跳过测试让套件通过
- 不得用空断言替换真实断言
- 如需修复生产代码，而非修补测试来变绿]

自调节节奏。每轮迭代后，执行检查命令，读取输出，仅在退出条件未满足时继续。退出条件通过或达到最大轮次时停止。每轮给出简短状态更新。

## 2 Interval 型（定时轮询）

/loop [时间间隔] 启动「[Loop名称]」循环。

- 目标：[持续监控目标]
- 每轮之间执行：[检查命令]
- 退出条件：[退出条件，或持续运行无终点]
- 第一步：[初始检查操作]
- 护栏：[同上]

## 3 Event 型（事件门禁）

安装并运行「[Loop名称]」循环。

- 目标：[拦截或放行的条件]
- 每轮之间执行：[检查命令]
- 退出条件：[放行条件]
- [需安装钩子配置文件：.claude/settings.json]
- 护栏：[同上]

→ Interval 型和 Event 型不需要退出条件——它们的节奏已被外部事件或时间间隔锁定。

## 核心句型

整个语料库收敛为「跑 X 直到 Y 通过」——定义动作（X）、定义验证（Y）、定义停止条件（通过）。这就是 Agent 循环的原子句法。
