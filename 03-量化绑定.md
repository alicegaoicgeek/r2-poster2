# 3 量化绑定 覆盖率不能撒谎

R2- VPlan 五层方法论 · 第三层

每条 covergroup 必须有一个 checker。Coverage 是管道，Checker 是阀门——没有阀门，什么数字都能进来。

## 铁律 · 三种典型违反 · 解法

**Coverage ≠ 正确性。Coverage + Checker = 正确性。**

分家：coverage和checker团队各干各的。失联：命名不一致无法对号入座。分层断裂：coverage模块级收集，checker系统级运行。

解法：r2-vplan 每条covergroup生成时自动预留checker接口。PSS的covergroup+exec body同写一个action，语法层面杜绝分家。

## 五种假绿 · 覆盖率100%时的盲区

| # | 名称 | 说明 |
|---|------|------|
| 01 | 贪婪采样 | 采样在checker判定前触发 |
| 02 | 条件融合 | 互斥条件用逻辑或合并在一个bin |
| 03 | 寄存器遍历 | 遍历地址从未验证读写后的功能行为 |
| 04 | 抽象层混淆 | 事务层绿了，信号级波形未查 |
| 05 | 脱离上下文 | 隔离环境覆盖，系统上下文未跑 |

## 五级里程碑 · 多媒体显示子系统 · 每级有具体百分比门槛

| 阶段 | 目标 | 放行标准 |
|------|------|----------|
| Pre-Alpha | — | Clock & Reset验证。基础环境就绪。 |
| Alpha | 60% | 寄存器接口全部通过。基础功能跑通。 |
| Beta | 80%+ | 全内存映射跑通，代码冻结前基本功能达标。 |
| Beta+Verif | 90%+ | RTL冻结后全覆盖冲刺。Error/Corner不可为0.0%——12个错误测试项一个没通过，项目还在推进，这就是MECE盲区。 |
| Final | 100% | 签核四件套：Integration+Usecase+Performance全通 + Toggle达标 + GlueLogic CodeCov 100% + Bug趋势收敛下降 |

## Checker 的参考基准 · 不同 IP 类型的金模拿什么比对

| IP类型 | 金模/Checker参考 | 能否逐拍比对 |
|--------|------------------|--------------|
| CPU/XPU/ISS (Spike/QEMU/Fast Models) | Lockstep逐指令：PC+寄存器+内存三路diff | ☑ |
| 算法类 C/MATLAB/浮点定点模型 | bit-true标准接口，算法团队提供 | ☑ |
| 接口与互联类 VIP参考模型 + Python协议合规VIP，性能靠Python模型+仿真拟合闭环，功能正确靠FPGA+真卡互操作 | | |
| 控制系统 Simulink/浮点/定点模型 | 阶跃响应+波特图+奈奎斯特分析。HDL Coder翻译成参考RTL | ☑ |
| 全芯片类 子系统级参考模型 + Boot固件 | 子系统复用验证 + 场景集成测试 + Boot/Software驱动。交互错误靠集成测试抓 | |
