01  三层接口哲学
--------------------------------------------------------------------------------

  +------------------+   +------------------+   +------------------+
  | 管道 PIPE        |   | 阀门 VALVE       |   | 翻译 TRANSLATOR  |
  | 信息通道结构     |   | 信息边界控制     |   | 跨层透明承诺     |
  +------------------+   +------------------+   +------------------+
  | 信息在独立通道上 |   | 什么能过界，     |   | 同一份意图，     |
  | 走，互不阻塞。   |   | 什么被挡住。     |   | 多套物理实现。   |
  | 三条独立通道：   |   | 接口冻结后只有   |   | 上层只面对接口， |
  | 命令、数据、响应 |   | 约定的信息允许   |   | 不感知底层差异。 |
  |                  |   | 跨层。           |   |                  |
  +------------------+   +------------------+   +------------------+


02  五类 IP 验证落点
--------------------------------------------------------------------------------

  IP类型          | 切分                    | 管道                      | 阀门                        | 翻译
  ----------------+-------------------------+---------------------------+-----------------------------+----------------------------------
  CPU / XPU       | Pipeline 阶段           | AMBA 总线 = 信息通道      | MMU 页表权限 + 中断使能     | C→汇编→机器码，ISS Lockstep 比对
                  | (取指/译码/执行/写回)   |                           |                             |
  ----------------+-------------------------+---------------------------+-----------------------------+----------------------------------
  算法类          | 数据流/控制流分离，     | Mem 进 Mem 出             | 精度限制 = 合法数据范围     | 浮点→定点→RTL，bit-true 对齐
                  | 标准接口层              | 归一化定点数据流          |                             |
  ----------------+-------------------------+---------------------------+-----------------------------+----------------------------------
  接口与互联类    | 协议分层 +              | TX/RX 双通道 +            | LTSSM 状态机 + QoS 仲裁     | VIP 翻译为事务层 + 性能模型闭环
                  | Master/Slave 拓扑       | 多链路并发                |                             |
  ----------------+-------------------------+---------------------------+-----------------------------+----------------------------------
  控制系统        | 控制环路/功率级/        | 反馈信号线 =              | PID 限幅 + 饱和保护         | Simulink→HDL Coder→RTL
                  | 采样器分离              | 闭环信息通道              |                             |
  ----------------+-------------------------+---------------------------+-----------------------------+----------------------------------
  全芯片类        | 按子系统切分，          | 子系统间数据流 =          | 子系统接口契约 =            | 子系统验证→全芯片集成
                  | 接口契约冻结            | 跨层通道                  | 跨边界控制                  |


03  参考基准
--------------------------------------------------------------------------------

  IP类型          | 参考形态                          | 替代 / 补充策略
  ----------------+-----------------------------------+-----------------------------------------------
  CPU / XPU       | ISS 指令集模拟器                  | Lockstep 逐指令 + TraceLog + printf 自检
  算法类          | C / MATLAB 浮点定点模型           | bit-true 标准接口，需定点化转换
  接口与互联类    | VIP 参考模型 + Python 性能模型    | 协议合规靠 VIP + 性能靠 Python 模型 + 真卡互操作
  控制系统        | Simulink 浮点 / 定点模型          | 频域 + 时域分析 + HDL Coder 生成参考 RTL
  全芯片类        | 子系统级参考模型 + Boot 固件      | 子系统复用验证 + 场景集成 + Boot/Software 驱动
