# R2-三刀法全景图

项目/特性切分框架 — 切 DITL 数据流, 不切模块

## 试错成本 = 影响范围 × 反馈代价 × 对齐损耗

| | 第一刀 | 第二刀 | 第三刀 |
|---|---|---|---|
| 因果链 | 1 第一刀：切出互不触发回归的块 | 2 第二刀：每个块内短迭代自闭环 | 3 第三刀：把闭环的块纵向拼成端到端交付 |
| 别名 | Backbone / 横轴叙事切 | Feedback Loop / 纵轴闭环 | Release Slice / 端到端对齐切 |
| 控什么 | 影响范围 — 改一处只坏一处 | 反馈代价 — 改完RTL到知道对错的时间 | 对齐损耗 — 多特性拼合时的接口摩擦 |
| 做什么 | 多视角切出独立特性块（功能+接口+数据流+寄存器） | 每个特性建短闭环自验，标必要/增强/未来裁 MVP | 短闭环特性按依赖组合成版本，解决拼合时的数据/时序/资源冲突 |

## 实例贯穿

### 第一刀: 特性地图

### 第二刀: 短闭环（以 LDPC 为例）
LLR输入→分层迭代→硬判决→比Golden Model
必要/增强/未来 → 收LLR迭代 / 多引擎并行 / QoS控制
Oracle: 机械裁判（Golden Model / Formal）

### 第三刀: 拼合对齐
R1: Boot+UART拼合，对齐点: 波特率? GPIO冲突?
R2: LDPC+信道+HARQ，对齐点: LLR格式? FIFO?
DoD: 上电→LDPC译码→HARQ反馈NACK→重传→Pass

## 关键区分
- **第二刀 ≠ 端到端** — 单特性内部自闭环，分钟/小时级反馈
- **第三刀 = 端到端** — 多特性拼合 + 对齐摩擦点，端到端可演示 DoD
