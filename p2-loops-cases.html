01
GOAL
目标制
Goal-based · 自动提升测试覆盖率
/goal · 最多 7 轮
START PROMPT
/goal 把当前项目测试覆盖率从68%提到85%以上，最多7轮 使用 SKILL.md 验证每轮变更 每轮结束后跑 npm test --coverage 并检查报告
SKILL.MD 关键行
  • 必须有新测试文件或修改
  • 覆盖率报告显示 ≥85%
  • 无新 failing test
启动：直接粘贴上面 prompt · 确定性阈值让评估模型知道何时收工
02
TIME
时间制
Time-based · 每日代码健康检查
/loop 1d · 22:00
START PROMPT
/loop 每天 22:00 检查所有未解决 issue + PR 对每个高优先级 issue 自动生成修复建议并创建 draft PR 停止条件：所有优先级 issue 处理完毕或超过10个
SKILL.MD 关键
  • 优先级：label=bug + assignee=me
  • 修复建议必须含代码 diff
  • + 测试命令
启动：/loop 1d + 上面 prompt · 间隔匹配「每日变化」频率，不空转
03
TURN
回合制+
Turn-based + SKILL · 代码审查 + 自动修复
SKILL 清单 · 你只确认
START PROMPT
对当前打开的 PR 进行完整审查，使用 SKILL.md 逐条检查 发现问题立刻自动修复并 commit 我只在最后确认
SKILL.MD 模板
  • 1. 安全漏洞 (npm audit)
  • 2. 性能问题 (console.time)
  • 3. 可读性 (命名+注释)
  • 4. 测试覆盖
  • 验证：eslint + vitest
启动：新建 SKILL.md 模板 → 粘贴 prompt · 手动验证步骤编码进文件
04
PRO
主动式
Proactive · Bug Triage + 自动修复流水线
/schedule 3h · auto
FULL START PROMPT
/schedule 每3小时 + dynamic workflow + auto mode 1. 检查 GitHub 新 issue (label=bug) 2. /goal 用 SKILL.md 复现 + 定位文件 3. 自动生成修复 commit + draft PR 4. 用 reviewer agent 检查 停止：issue 关闭或 3 轮未解决
组合件
  • /schedule 掐点巡检
  • /goal 定复现终点
  • workflow 编排步骤
  • auto mode 免审批
  • 独立 reviewer 把关
启动：整段粘贴 · 对应 P1 第四档「全程不用你管」+ 独立 review 质量线
05
MIX
混合最强
混合型 · Oh-My-Pi / Terminal Agent 最佳实践维护
/schedule 08:30 每日
START PROMPT（定制）
/schedule 每天 08:30 1. 拉取 Oh-My-Pi 最新文档 2. /goal 对比我的 best-practices.md，找出过时或缺失部分 3. 用 SKILL.md 验证新实践（必须有命令+输出示例） 4. 自动更新文档 + commit + 通知我
为何算「最强」
  • 时间制：每日定点
  • 目标制：diff 文档缺口
  • SKILL：命令+输出必有
  • 产出：更新+commit+通知
  • 知识库自进化，人只收报
启动：粘贴 prompt · schedule × goal × skill 三件套，适合「文档/实践」类持续资产
