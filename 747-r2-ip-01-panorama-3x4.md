## L2 对不对 | 接不进 · 空壳 = 契约

**接口契约（先签字，后编码）**

| Stub（结构占位） | Behavioral（行为占位） | Golden（参考标杆） |
|---|---|---|
| REQ → IP_SHELL → RSP · 端口/时序/参数占位 · 无功能，只管结构 | REQ → IP_BEHAV → RSP · 行为级模型 · 返回合法响应/约束 | REQ → IP_GOLDEN → RSP · 黄金模型（参考）· 判定正确性基线 |

### 契约清单（必须项）
- 接口信号/时序/协议
- 参数/配置/地址映射
- 约束/假设/覆盖目标
- 版本/变更/兼容策略
- 验收标准/退出条件

## L3 四方同一套 case 跑 EDA

### 四方同一套 case（同源·同配·同标·同分）
- **IP团队**：开发/自测 · 提供模型
- **SoC团队**：集成/验证 · 问题闭环

- **DFT/工具团队**：扫描/DFT · 工具脚本

**统一：case库 / 约束 / 环境 / 指标 / 报告（可追溯）**

### EDA门禁（四道闸）
1. 编译过
2. 仿真过
3. 覆盖过
4. Lint/CDC/DFT过

## L4 shell钉死 · 只换DUT

### 稳定集成壳（IP_SHELL）
- TOP / IP_SHELL：周边/系统逻辑
- 只换 DUT：DUT_v0 → DUT_v1 → … → DUT_vN
  - 壳不动，接口不变，验证环境不变
  - 版本替换，快速验证，风险可控

### 收益
- 减少集成震荡
- 避免连锁重改
- 加速并行推进

### 壳清单（必须项）
- 端口/时序/复位
- 参数/寄存器映射
- 时钟/电源/低功耗
- DFT/可测性插桩

## L5 短回路 · 止于功能 · CI ≠ 流片

### 短回路验证（快速·充分·可信）
提交（CI触发）→ 构建（编译/仿真）→ 运行（case/仿真）→ 检查（覆盖/断言）→ 归档（报告/波形）→ 止于功能（非流片）

**CI目标：功能正确 / 接口稳定 / 约束满足 / 覆盖达标**

### CI门禁（默认红线）
- ✗ 功能失败 = 红
- ✗ 断言失败 = 红
- ✗ 覆盖不足 = 红
- ✗ Lint/CDC = 红
