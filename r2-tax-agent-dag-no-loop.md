# R2-税务 Agent 纠错学习 DAG：单次执行无环

## 主链路

1 启动 → 2 读取用户资料 → 3 理解税务规则 → 4 使用工具填写表格 → 5 置信度判断

### 分支一：高置信度
5 → 6a 提交申报 → 12a 归档完成

### 分支二：不确定 / 错误
5 → 6b 人类专家介入（同时触发）6c 记录 traces：完整执行过程  
→ 7 专家注释修正：指出错在哪步，正确做法是什么  
→ 8 Codex 自改进：改提示词 / 工具定义 / 工作流代码  
→ 9 生成 eval 测试：针对有测试约束后续改动  
→ 10 人工审核确认：工程 + 业务专家判断能否进生产  
→ 11 更新系统  
（虚线 11 → 2：非 DAG 内部边，表示新版本下次执行入口）

## 关键说明

- **5 置信度判断**：不确定时知道停下来让人类专家介入
- **6b / 6c 并行**：专家介入时，系统同时记录完整 traces
- **7 专家注释**：专家不是简单打差评，而是指出哪一步错了、正确做法是什么
- **8 Codex 自改进**：拿失败案例、专家修正和 eval 去改提示词、工具定义、工作流代码
- **9 生成 eval**：补上更有针对性的测试
- **10 人工审核**：哪些改动能进生产，仍然需要工程和业务专家判断
- **11 → 2 虚线**：不是 DAG 内部的边，只表示新版本下次执行入口

## 结论

- 两条互斥分支
- 一条通向完成
- 一条通向系统更新
- **没有回环。**
