# 5 · 五词榫卯

R2-ONTOLOGY · 词

卵口不执行只定义接受形状。榫头不定义边界只在边界内运动。两者互补——缺一为零。

## 01 Harness = 卵口
凹进去的空间——不执行，只定义接受形状。编排 agent/parallel/pipeline/while，定义边界和停止条件。挖掉的东西比留下的东西更关键。

## 02 Loop = 榫头
凸出的运动体——在卵口内自主执行。  
Intent→Context→Action→Observation→Adjustment。  
有缝可见，碰壁即停。

## 03 Skill = 榫头形状
决定了咬合的具体方式。燕尾榫 ≠ 直角榫——specx 的咬合方式与 verismart 不同。形状不是内容，是接口。

## 04 CLI = 插入通道
力传导的物理接触面——Bash/Grep/Read/Git 是榫头触碰卵口壁的点。没有通道，形状永远碰不到边界。

## 05 LLM = 木材弹性
材料的变形与恢复能力——决定能适应多复杂的卵口形状而不折断。弹性不是智能，弹性是容错空间。
