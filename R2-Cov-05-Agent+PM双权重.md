# Agent 和 人类 同一个函数的两个权重源

R2-Cov · 回归筛选  
05 / 06

## 不是二选一，是叠加

| 维度 | Agent | 人类 |
|------|-------|------|
| 输入 | 代码变更 + 测试历史 | 需求变更 + 业务影响 |
| 处理 | 蜂群并行查证 | 人类 优先级判断 |
| 输出 | 结构化判定表 | 优先级排序列表 |
| 反馈 | 查证结果回写 | 结果调整优先级 |
| 权重源 | 覆盖率 + 历史数据 | 业务上下文判断 |
| 优势 | 大规模、一致性 | 上下文、可解释 |
| 盲区 | 无业务上下文 | 样本量小、不一致 |

**叠加方案：基准分（Agent 查证 0–100）+ 人类 覆盖分（+50）= 最终排序**  
"客户下周 demo low power → low power 测试无条件进 smoke，+50 分。"

## Agent 提供基准，人类 提供覆盖。不是二选一，是叠加。

Agent 解决大规模一致查证，人类 解决上下文信息补位。同一个函数签名：变更 → 排序 → 子集 → 执行 → 反馈。
